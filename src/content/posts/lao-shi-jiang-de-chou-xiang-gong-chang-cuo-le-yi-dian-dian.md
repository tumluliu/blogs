<span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">    他也有讲错的时候，虽然只是一点点，但还是被我抓到了，呵呵。而且通过对错误地方分析可以知道，错因是他对<span lang="EN-US">OOD基本</span>原则的理解以及对<span lang="EN-US">Abstract Factory</span>的理解似乎还不是十分到位。\
<span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">    看得出，他讲《设计模式》的蓝本是英文版的<span lang="EN-US">GoF</span>《<span lang="EN-US">Design Patterns</span>》，课件上都是清一色的英文，对每一个<span lang="EN-US">DP</span>的介绍模式也都是按照<span lang="EN-US">GoF</span>那本书来的。但是他今天在讲到抽象工厂局限性的时候，用红色字体打出：\
<span lang="EN-US" style="FONT-SIZE: 11pt; COLOR: red; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">    difficult to ... new kinds of objects</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">（<span lang="EN-US">p.s.</span>原话记不清了）\
<span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">    然后解释说：抽象工厂的局限性就在于当有新的产品线<span lang="EN-US">3</span>需要加入系统的时候，不但要新建继承自<span lang="EN-US">AbstractProcuctA</span>的<span lang="EN-US">ProductA3</span>类，新建继承自<span lang="EN-US">AbstractProcuctB</span>的<span lang="EN-US">ProductB3</span>类，“</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">这倒没有什么”</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">，而且还要新建一个集成自<span lang="EN-US">AbstractFactory</span>的<span lang="EN-US">ConcreteFactory3</span>，“</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">这就不行了”</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">。从他的语气中可以知道他认为新建类，特别是要新建一个具体工厂是“</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">不好”</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">的。\
<span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">    然而就我目前对<span lang="EN-US">DP</span>的理解，情况却不是这样的。当有产品线<span lang="EN-US">3</span>需要加入系统时，正好是抽象工厂显示威力的时候。新增加的三个派生类<span lang="EN-US">ProductA3</span>、<span lang="EN-US">ProductB3</span>和<span lang="EN-US">ConcreteFactory3</span>很好的展示了抽象工厂模式符合“</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">开放－封闭原则”</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">的特点</span><span lang="EN-US" style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA; mso-fareast-font-family: 宋体; mso-ascii-font-family: 宋体">——</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">新的需求没有对原有的设计和结构，特别是<span lang="EN-US">AbstractFactory</span>和<span lang="EN-US">AbstractProductX</span>接口做任何的修改，只是对现有系统进行了扩展，就实现了新的需求。而对“</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">增加新的产品线<span lang="EN-US">/</span>族<span lang="EN-US">/</span>系列”</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">这个轴线上的需求变化的应对自如也正是抽象工厂的力量所在。\
<span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">    那么对于<span lang="EN-US">GoF</span>书中的那句“<span lang="EN-US">difficult to ... new kinds of objects”</span>怎么理解呢？\
<span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">    抽象工厂的局限性应该是当需求变化的轴线在于“</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">增加组成系统的新产品<span lang="EN-US">/</span>对象”</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">的时候，就力不从心了。\
    这里就拿老师上课举的例子来说明。他的例子是构建一个迷宫游戏（<span lang="EN-US">Maze Game</span>），组成迷宫（<span lang="EN-US">Maze</span>）的基本元素有三个</span><span lang="EN-US" style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA; mso-fareast-font-family: 宋体; mso-ascii-font-family: 宋体">——</span><span style="FONT-SIZE: 11pt; COLOR: black; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">房间（<span lang="EN-US">room</span>）、墙壁（<span lang="EN-US">wall</span>）和门（<span lang="EN-US">door</span>）。游戏分为不同的</span>~~<span style="FONT-SIZE: 11pt; COLOR: #969696; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">难度等级，比如<span lang="EN-US">easy/normal/hard</span>，对应于不同的难度，房间、墙壁和门是不同的，比如<span lang="EN-US">hard</span>模式的门可能是带有密码锁的。其实这个例子看起来并不十分恰当，因为通常应该觉得迷宫的难度等级应该按照其内部的复杂度来划分，而不是按照他的这种方式来划分，这个暂且不管，就先用他的划分方式吧。</span>~~<span style="FONT-SIZE: 11pt; FONT-FAMILY: 宋体; mso-hansi-font-family: 'Times New Roman'; mso-bidi-font-family: 宋体; mso-ansi-language: EN-US; mso-fareast-language: ZH-CN; mso-bidi-language: AR-SA">场景，比如丛林迷宫、楼房迷宫等等。在这些不同的场景下，房间、墙壁和门的风格当然也是不同的，所以这里应用抽象工厂是一个合适的选择。\
</span></span></span></span></span></span></span>
